Functionality

Visual Studio Code

I originally created the trading site application as part of a problem set in CS50, this was done within CS50's own IDE.

It was written in Python 3 and used the Flask framework. The database structure used was SQLite.

I then setup Visual Studio Code and set about transferring over the application. After this I created a blog website and then combined both and added extra functionalities.

Virtual Environment

I used venv, which is a built-in virtual environment for Python 3.

Database

The database structure I used was SQLAlchemy (as opposed to SQLite in my original app). I built a database from scratch with User and Post tables using tutorials and then modified them to suit what I needed. I created a transaction table to keep track of the bought and sold shares.

User Setup

There is a registration page and a login page for the site. I used Flask Login to handle users. I added a forgot password function whereby the user is emailed a link to reset their password. I used environment variables saved locally on my system in place of the email address and password used to send the password reset email (a secret key and database URI are also saved this way).

The user will be notified if the account info is not valid when registering or logging in.

The user can also change their account info and password in the Account tab.

Flask Admin



In order to have administration rights for the site I used Flask Admin. This allows the administrators to access the users and posts tables via the user interface (tab in the navbar when logged in as 'admin', see above clip) and edit/delete entries as they wish. I did not allow access to the transaction table as this should not be accessible to anyone.



There is a role column in the user table which is set to default as 'user'. I set my own role to 'admin', only an admin can access the user table in the Admin interface and change the role of a user.



I created a link within the Admin interface to direct back to the main website.

Blueprints

I separated out some of the different functionalities into 4 blueprints:

  1. Main
  2. User
  3. Posts
  4. Finance

Although this project is relatively small this is good practice as large projects could get very messy.

Error Pages

Customised error pages are included which will indicate the potential problem based on what the user was trying to do.

Search Bar

I added a search bar for the blog section of the website. This searches the title and content of each post and displays the relevant post on a results page. This was created using Flask Whoosh.

Application Initialisation

In order to create the application I set up a function called create_app. This function is called on once the program is ran. The configuration settings are saved to a class in a different .py file and this is passed into the create_app function.

Hosting

The website is hosted using a Linode server. I choose the option with the lowest available memory as this was the cheapest and the site requires very little memory. I did not purchase a domain name as there is no need.
